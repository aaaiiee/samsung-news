# Interaction Protocol

## 질문 기본 규칙

- 인터뷰 단계는 반드시 한 번에 `1개` 질문
- 워크시트 이후 단계는 한 번에 `3개 이하` 질문
- 현재 단계에 꼭 필요한 질문만
- 앞 단계에서 확보한 답변은 재사용
- 인터뷰의 핵심 항목이 비면 다음 단계로 넘어가지 않음
- 충분히 질문했지만 사용자가 현재 답할 수 없는 항목만 `[보류: 이유]`로 남김
- 비핵심 검증 항목이나 추후 확인 항목만 `[확인 필요]`로 남김
- 현재 흐름과 무관한 과거 운영 체계는 설명하지 않음

## 인터뷰 정본

- 인터뷰 시작 전 `python .claude\skills\_education_skills_중급\automation-pipeline-design-eduv-중급\scripts\find_gpters_study_docs.py --stage interview` 결과를 먼저 확인합니다.
- 인터뷰 단계의 실제 질문 깊이와 순서는 문서 매핑기에서 `interview` 역할로 찾은 현재 문서를 우선합니다.
- 이 문서는 질문 운영 규칙을 요약한 문서이며, 인터뷰를 4개 항목만 확인하는 간이 체크리스트로 축소하면 안 됩니다.

## 단계별 질문 우선순위

### 인터뷰

- 누가 관여하고 누가 수혜자인가
- 현재 어떤 고통과 병목이 있는가
- 자동화 범위와 사람 개입 지점은 어디인가
- 언제, 어떤 조건에서 시작되는가
- 무엇을 입력받고, 어떤 검증이 필요한가
- 무엇을 출력하고 누구에게 전달하는가
- 중간 처리 흐름과 실패 시 멈춤 지점은 어디인가
- 예외, 승인, 저장 위치, 로그, 실행 환경 제약은 무엇인가

### 워크시트

- 앞 단계 답변 중 빈칸은 무엇인가
- 이미 준비된 자료와 앞으로 모아야 할 자료는 무엇인가
- 워크시트만 읽어도 제3자가 업무를 이해할 수 있는가
- 핵심 빈칸은 추가 인터뷰가 필요한가, 아니면 `[보류: 이유]`로 넘길 수 있는가

### 자동화 설계

- 사람 검토 지점은 어디인가
- 실패 시 어디에서 멈춰야 하는가
- 중복 방지와 결과 저장은 어떻게 처리하는가
- 워크시트 핵심 섹션이 약하면 어떤 항목을 다시 인터뷰해야 하는가

### 도구 연결

- 현재 작업공간에서 찾은 스킬은 무엇인가
- 지금 바로 연결 가능한 스킬은 무엇인가
- 추가 설정이 필요한 도구는 무엇인가
- 없는 기능은 무엇인가
- 설계가 불안정해서 아직 연결 판단을 미뤄야 하는 항목은 무엇인가

### 오케스트레이션 실행 및 결과 확인

- 실행에 필요한 입력 자료가 준비되어 있는가
- 지금 가능한 범위는 실제 실행인가 드라이런인가
- 결과를 누가 어떻게 검토하는가
- 아직 실행하면 안 되는 이유가 있다면 무엇인가

## 단계 전환 규칙

- 인터뷰에서 필수 항목이 비어 있으면 워크시트로 넘어가지 않습니다.
- 워크시트 작성 중 핵심 공백이 발견되면 자동화 설계로 넘어가지 않고 인터뷰 보완 질문을 먼저 합니다.
- 자동화 설계 이후에도 핵심 공백이 발견되면 인터뷰 또는 워크시트 단계로 되돌아갑니다.
- 실행 단계는 입력 자료와 전제조건이 확인되었을 때만 실제 실행하고, 그렇지 않으면 실행 보류 메모 또는 드라이런 계획으로 남깁니다.

## 금지

- 한 번에 긴 설문처럼 묻는 것
- 문서 전체 빈칸을 한 번에 요구하는 것
- 같은 질문을 반복하는 것
- 인터뷰를 얕게 끝내고 다음 단계로 서둘러 넘어가는 것
- 아직 없는 도구를 있다고 가정하는 것
- 복잡한 상태관리부터 묻는 것
