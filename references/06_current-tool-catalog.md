# Skill Discovery Template

## 사용 원칙

- 먼저 `현재 작업공간에 실제로 있는 스킬`로 해결할 수 있는지 봅니다.
- 없는 기능은 `그레이 영역`으로 표시합니다.
- 그레이 영역은 실패가 아니라 `추가 제작 제안`입니다.

## 1. 작업공간 스킬 찾기

PowerShell 예시:

```powershell
Get-ChildItem -Recurse -Filter SKILL.md .\.claude\skills
```

필요하면 현재 작업공간 전체도 추가 확인:

```powershell
Get-ChildItem -Recurse -Filter SKILL.md .
```

## 2. 각 스킬에서 확인할 것

각 `SKILL.md`에서 아래를 확인합니다.

- 스킬 이름
- 설명
- 입력 형식
- 출력 형식
- 실행 스크립트 유무
- 현재 업무에 바로 쓸 수 있는지 여부

## 3. 도구 연결 판단표

아래 기준으로 정리합니다.

| 분류 | 의미 |
|---|---|
| 즉시 사용 가능 | 지금 자료와 설정으로 바로 쓸 수 있음 |
| 추가 설정 필요 | 인증, 패키지, 경로 확인 등이 필요함 |
| 그레이 영역 | 현재 스킬이 없거나, 있어도 요구 기능을 직접 처리하지 못함 |

## 4. 도구 연결 출력 형식

아래 형식으로 정리합니다.

```text
도구 연결표:
- 자동화 단계: [예: 회의록 분석]
- 찾은 스킬/도구: [스킬명 또는 없음]
- 사용 가능 여부: [즉시 사용 가능 / 추가 설정 필요 / 그레이 영역]
- 판단 근거: [SKILL.md 설명 요약]
- 예상 입력: [입력 자료]
- 예상 출력: [원하는 결과]
```

## 5. 그레이 영역 출력 형식

그레이 영역이 생기면 아래 형식으로 남깁니다.

```text
그레이 영역:
- 필요한 기능: [무엇이 필요한가]
- 왜 필요한가: [현재 자동화 단계에서의 역할]
- 예상 입력: [입력 자료]
- 예상 출력: [원하는 결과]
- 제안 스킬 이름: [새 스킬 이름 후보]
```
