# Document Classification

## 1. 참고 문서

- 내부 허브 문서
- 전체 흐름 참고 문서
- 사용 설명서
- 예시 문서

역할:

- 사용자가 전체 구조를 이해하도록 돕는 문서
- 직접 제출하는 최종 산출물은 아님

## 2. 인터뷰/정리 문서

- 인터뷰 참고 문서
- 워크시트 참고 문서

역할:

- 사용자의 업무를 드러내고 구조화하는 문서

## 3. 자동화 설계 문서

- 내부 자동화 설계 템플릿
- 자동화 설계 예시 문서

역할:

- 내부 템플릿을 참고해 사용자용 `자동화 설계 초안`을 만드는 문서
- 사용자에게는 내부 용어보다 `자동화 설계` 표현을 우선 사용

## 4. 실행 산출물

이 스킬이 새로 만드는 산출물:

- `automation_design.md`
- `tool_connection.md`
- `orchestration_run.md`
- `run_result.md`

역할:

- 현재 작업공간에서 실제로 찾은 도구와 그레이 영역을 구분
- 실제 실행 순서와 결과를 확인
