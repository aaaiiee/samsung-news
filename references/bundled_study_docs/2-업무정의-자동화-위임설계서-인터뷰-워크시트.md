## ✍️ 업무 위임 설계 워크시트
**나의 AI팀에게 일을 맡기기 위한 업무 정의 작성지**

> 📌 작성 규칙
> - 완벽하지 않아도 됩니다. 지금 당장 떠오르는 것을 씁니다.
> - AI에게 질문하여 답변을 받고 수정하는 과정 자체가 작업입니다.
> - 총 소요 시간: 약 1시간 ~ 1시간 30분
> - 이 문서는 `초안 작성지`입니다. 작성 완료 후 자동화 설계 단계의 입력으로 사용합니다.
> - 참고 문서 위치와 결과 문서 위치를 분리합니다. 완성본은 `C:\501_aiworks\10_working\120autoproject\<120x_업무주제_automation>\output`에 저장합니다.

| 이 업무의 자동화는 누구를 위한 것인가요? | □ **나 혼자의 업무** (권장) / □ 여러 사람의 업무 |

> 💡 **권장**: 처음에는 **나 1명의 업무**를 자동화하는 것부터 시작하세요.
> 다수의 수혜자를 위한 자동화는 업무 정의가 매우 복잡해지므로, 1인 자동화를 먼저 완성한 후 확장하는 것이 현실적입니다.

## Step 1️⃣ 참여자·수혜자 매핑

> "이 업무에 관여하는 모든 사람과 각각의 역할은 무엇인가?"

### 1-A. 업무 참여자 전체 식별

```text
[요청자] 이 업무를 시작시키는 사람은?
→

[실행자] 이 업무를 직접 수행하는 사람은?
→

[수혜자] 이 업무의 결과물을 받는 사람은? (복수 가능)
→

[의사결정자] 중간 확인/승인이 필요한 사람은?
→

[참조자] 통보/CC가 필요한 사람은?
→
```

### 1-B. 참여자 간 현재 업무 흐름

현재 누가 누구에게 무엇을 주고받는지 정리합니다.

| From (보내는 쪽) | To (받는 쪽) | 주고받는 것 | 빈도 | 형태 (구두/메신저/이메일/파일) |
| --- | --- | --- | --- | --- |
| | | | | |
| | | | | |
| | | | | |
| | | | | |

### 1-C. 수혜자별 상세 분석

> 디폴트는 **1명**(나 자신)입니다. 수혜자가 복수인 경우에만 추가 작성합니다.

**수혜자 #1 (필수):**

| 항목 | 내용 |
| --- | --- |
| 수혜자 (이름/역할) | |

| 결과물을 받는 채널/형식 | |

**수혜자 #2:** (여러 사람 선택 시에만 작성)

## Step 3️⃣ AI팀 업무 위임 선언서

> "AI팀이 이 업무를 완수했을 때의 장면을 지금 씁니다"

### 3-A. 자동화 범위 선언

각 단계별로 자동화 유형을 지정합니다.

```text
🟢 완전 자동화: AI가 100% 처리. 사람은 결과만 확인.
🟡 부분 자동화: AI가 실행하되, 사람의 확인/검증/수정 필요.
🔴 사람 전용:   사람이 반드시 직접 판단/실행.
```

| 업무 단계 | 현재 수행자 | 자동화 유형 (🟢/🟡/🔴) | 비고 |
| --- | --- | --- | --- |
| | | | |
| | | | |
| | | | |
| | | | |
| | | | |

### 3-B. 사람 개입 지점 (Human-in-the-Loop)

🟡 부분 자동화 단계에서 **사람이 무엇을 확인/결정하는지** 구체화합니다.

| 단계 | 사람이 확인할 내용 | 판단 기준 | 확인 후 행동 |
| --- | --- | --- | --- |
| | | | |
| | | | |
| | | | |

### 3-C. 위임 선언서 작성

이미 완성된 것처럼 **완료형**으로 씁니다.

```text
[AI팀 업무 위임 선언서]

업무명: [                              ]

나의 AI팀은 이제 이 업무를 담당합니다.

[자동화 범위]
- 완전 자동화(🟢): [                              ]
- 부분 자동화(🟡): [                              ] → 사람 확인 지점: [      ]
- 사람 전용(🔴):   [                              ]

[동작 방식]
내가 [시작 명령/시점]을 내리면,
AI팀은 [처리 과정]을 거쳐
[최종 결과물]을 [수혜자]에게 [전달 방법]으로 전달합니다.

[기대 효과]
이로 인해 나는 [구체적 효과]를 얻게 되었습니다.
```

## 🔍 자가 점검 체크리스트

작성을 마친 후 아래를 확인하세요:

```text
□ 참여자(요청자/실행자/수혜자/의사결정자)가 빠짐없이 식별되었는가?
□ 참여자 간 현재 업무 흐름(주고받는 것, 빈도, 형태)이 기록되었는가?
□ 수혜자별로 고통 분석이 각각 작성되었는가?
□ 자동화 범위가 🟢/🟡/🔴로 구분되었는가?
□ 🟡 부분 자동화에서 사람 개입 지점이 구체적으로 정의되었는가?
□ 대표 실행 명령과 명령 채널이 명확히 정의되었는가?
□ 입력 데이터의 소스/형태/항목이 정의되었는가? (템플릿은 별도 첨부 가능)
□ 출력 결과물의 형태/파일명/저장위치가 정의되었는가? (템플릿은 별도 첨부 가능)
□ 저장 위치/파일명 규칙/전달 방법이 정해졌는가?
□ 예외가 입력/처리/출력 단계별로 최소 3개씩 정의되었는가?
□ 에스컬레이션 정책에 무응답 시 처리가 포함되었는가?
□ (새로 추가) 각 처리 단계별로 사용할 도구 범주(명령 수신기/수집기/변환기/발송기/저장기 등) 또는 실제 도구가 지정되었는가?
□ (새로 추가) 시스템 연동 시 필요한 API 키 이름이나 환경변수(.env)가 명확히 정의되었는가?
□ (새로 추가) AI 분석/요약 시 필요한 모델/엔진 종류 또는 판단 기준이 정의되었는가?
□ (새로 추가) 실행을 위해 사전에 준비되어야 할 필수 입력 파일/데이터(todo list, 프롬프트 템플릿 등)가 확보되었는가?
□ 이 워크시트를 신입사원에게 보여줘도 이해할 수 있는가?
```
