# 사용자용 사용법 및 실행 설명서

## 이 문서의 역할

이 문서는 현재 중급 자동화 스킬을 실제로 사용할 때 무엇을 준비하고 어떻게 진행하는지 설명하는 매뉴얼입니다.

## 먼저 알아둘 것

### 참고 문서 위치

```text
<skill_root>\references\bundled_study_docs
```

### 실습 산출물 권장 위치

```text
<project_root>\
  source\
  output\
```

핵심 원칙:

- 참고 문서는 읽는 용도입니다.
- 결과물은 `output`에 저장합니다.
- 작업 중인 자료는 `source`에 둡니다.
- 복잡한 상태 폴더는 만들지 않습니다.

## 준비물

- 자동화하고 싶은 업무 1개
- 최근 실제 사례 1개 이상
- 입력 데이터 예시 1개 이상
- 원하는 결과물 예시 1개 이상

## 단계별 사용법

### 1. 인터뷰

AI에게 요청하는 예시:

```text
이 인터뷰 가이드를 기준으로 한 번에 질문 하나씩만 해줘.
내 답변을 들은 뒤 비어 있는 항목이 있으면 보완 질문을 해줘.
내가 더이상 답할 수 없는 항목만 [보류: 이유]로 남겨줘.
최종적으로 워크시트에 옮길 수 있게 핵심 답을 요약해줘.
```

완료 기준:

- 수혜자, 현재 고통, 자동화 범위, 시작 조건, 입력, 출력, 예외, 사람 검토 지점, 저장/전달을 말로 설명할 수 있습니다.
- 인터뷰의 큰 빈칸은 답변되었거나 `[보류: 이유]`로 정리되어 있습니다.

### 2. 워크시트

해야 할 일:

- 인터뷰 답변을 빈칸 양식에 옮깁니다.
- 핵심 빈칸은 그냥 넘기지 말고 보완 질문으로 다시 메웁니다.
- 사용자가 더이상 답할 수 없는 항목만 `[보류: 이유]`로 표시합니다.
- 작성한 파일은 `output`에 저장합니다.

완료 기준:

- 워크시트의 핵심 섹션이 모두 채워져 있거나 `[보류: 이유]`가 명시되어 있습니다.
- 워크시트만 읽어도 제3자가 업무를 이해할 수 있습니다.

### 3. 자동화 설계

AI에게 요청하는 예시:

```text
내 워크시트 내용을 바탕으로 자동화 설계 초안을 작성해줘.
Windows PowerShell 기준으로 쓰고 mac 혹은 linux 환경인 경우 해당 환경에 맞게 작성해줘.
워크시트 핵심 섹션이 비어 있으면 먼저 어떤 항목을 다시 인터뷰해야 하는지 알려줘.
없는 기능은 그레이 영역 또는 추가 개발 필요로 표시해줘.
```

완료 기준:

- 자동화 설계 초안이 생성되어 있습니다.

### 4. 도구 연결

해야 할 일:

- 현재 작업공간의 `SKILL.md`를 직접 찾습니다.
- 즉시 사용 가능 / 추가 설정 필요 / 그레이 영역으로 나눕니다.

완료 기준:

- 도구 연결표가 정리되어 있습니다.

### 5. 오케스트레이션 실행 및 결과 확인

해야 할 일:

- 입력 자료가 없으면 실행을 가장하지 말고 실행 보류 메모 또는 드라이런 계획을 남깁니다.
- 가능한 범위는 실제 실행과 드라이런을 구분해 기록합니다.

완료 기준:

- 실행/결과 확인 메모 또는 실행 보류 메모가 있습니다.

### 6. Agent 변환

해야 할 일:

- 오케스트레이션 실행 결과와 도구 연결표를 바탕으로 Agent 정의 파일을 작성합니다.
- 사람 개입 지점과 실제 호출 테스트 방법을 함께 정리합니다.

완료 기준:

- Agent 정의 파일과 최소 1회의 호출 테스트 방법이 정리되어 있습니다.

## 필수 제출물

- 인터뷰 답변 메모
- 워크시트
- 자동화 설계 초안
- 도구 연결표
- 실행/결과 확인 메모 또는 실행 보류 메모
- Agent 정의 파일

참고 제출물:

- 추가 보완 질문 메모

| Agent 정의 파일이 실행 순서를 반영하고 있는가 | |
