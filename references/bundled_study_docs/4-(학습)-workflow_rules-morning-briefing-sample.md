# 자동화 설계 예시: 명령 실행형 일일 브리핑

이 문서는 `예시/참고 문서`입니다.

사용자가 이 파일을 그대로 제출하는 것이 목적은 아닙니다.
목적은 자동화 설계의 구조와 톤, 검토 지점, 예외 처리 방식을 이해하는 것입니다.

## 위치 구분

```text
<project_root>\
  source\
  output\
```

## 예시

```text
# 자동화 설계 초안: 일일 브리핑 생성

## 1. 목적
명령이 들어오면 오늘의 브리핑 문서를 만들고 검토 후 전달한다.

## 2. 시작 트리거
- "오늘 브리핑 생성해줘"
- /run-briefing

## 3. 입력 수집 및 검증
- source 폴더의 뉴스 요약 자료
- source 폴더의 메모와 링크
- 필수 입력이 없으면 [확인 필요]로 멈춘다

## 4. 핵심 처리 단계
1. 자료를 읽고 핵심 주제를 3개로 정리
2. 초안 문서를 작성
3. 제목과 요약 문장을 정리

## 5. 사람 검토 지점
- 숫자와 고유명사 확인
- 표현 수위 확인

## 6. 저장 및 전달
- output\YYYYMMDD_daily_briefing.md
- 검토 후 메신저 또는 문서로 전달

## 7. 예외 처리
- 입력 자료가 비면 생성 중단
- 자료가 부족하면 [확인 필요]로 표시

## 8. 중복 방지
- 같은 날짜 파일이 이미 있으면 덮어쓰기 전에 확인

## 9. 현재 자동화 가능한 부분
- 자료 읽기
- 초안 작성
- 저장

## 10. [확인 필요] 항목
- 최종 전달 채널
- 최종 검토자
```

## 이 예시를 볼 때 확인할 것

- 목적이 한 문장으로 정리되어 있는가
- 시작 조건이 분명한가
- 사람이 확인해야 할 지점이 있는가
- 예외 처리와 `[확인 필요]` 항목이 남아 있는가
