# 도구 연결표: 삼성전자 뉴스룸 생활가전 뉴스 자동 브리핑

## 작업공간 스킬 탐색 결과

| 탐색 위치 | 결과 |
|-----------|------|
| `.claude/skills/` | 디렉토리 없음 |
| 루트 `SKILL.md` | 자동화 교육 스킬 (도구 연결 대상 아님) |

> 현재 작업공간에 직접 사용할 수 있는 실행 스킬이 없으므로, **Python 패키지 + 외부 API + 로컬 AI Agent**를 기준으로 도구를 연결합니다.

## 1. 도구 연결표

| 단계 | 자동화 작업 | 연결 도구 | 사용 가능 여부 | 판단 근거 | 예상 입력 | 예상 출력 |
|------|-------------|-----------|----------------|-----------|-----------|-----------|
| ① | 뉴스룸 기사 목록 수집 | `requests` + `BeautifulSoup4` | ✅ 즉시 사용 가능 | Python 표준 스크래핑 조합, pip 설치만 필요 | 뉴스룸 URL | 기사 목록 [{제목, URL, 날짜, 카테고리}] |
| ② | 생활가전 주제 필터링 | Python 내장 (`in`, 정규식) | ✅ 즉시 사용 가능 | 키워드 리스트 매칭, 추가 패키지 불필요 | 기사 목록 + 키워드 리스트 | 필터링된 기사 목록 |
| ③ | 폴백 판단 (기사 없을 때) | Python 조건 분기 | ✅ 즉시 사용 가능 | 단순 if/else 로직 | 필터링 결과 (빈 리스트 여부) | 최종 기사 목록 + 폴백 플래그 |
| ④ | 기사 본문 수집 | `requests` + `BeautifulSoup4` | ✅ 즉시 사용 가능 | ①과 동일 조합 | 기사 URL 목록 | 기사 본문 텍스트 목록 |
| ⑤ | AI 요약 생성 (3줄 + 해시태그) | **로컬 AI Agent** (Claude Code) | ✅ 즉시 사용 가능 | 현재 세션 AI가 직접 수행, 외부 API 키 불필요 | 기사 본문 텍스트 | 3줄 요약 + 해시태그 |
| ⑥ | 텔레그램 메시지 조립 | Python 문자열 처리 (f-string) | ✅ 즉시 사용 가능 | 추가 패키지 불필요 | 요약 데이터 + 메시지 템플릿 | 포맷팅된 메시지 문자열 |
| ⑦ | 텔레그램 전송 | `python-telegram-bot` | 🔧 추가 설정 필요 | pip 설치 + 봇 토큰/채팅 ID 설정 필요 | 메시지 문자열 | 전송 성공/실패 결과 |
| ⑧ | 노션 DB 기록 | `notion-client` | 🔧 추가 설정 필요 | pip 설치 + 노션 API 키/DB ID 설정 필요 | 기사 데이터 + 요약 | DB 행 추가 결과 |
| ⑨ | 스케줄 실행 (자동 트리거) | Mac `launchd` 또는 `crontab` | 🔧 추가 설정 필요 | OS 내장, plist 또는 crontab 작성 필요 | cron 표현식 (0 8 * * *) | 매일 오전 8시 스크립트 실행 |
| ⑩ | 수동 트리거 (텔레그램 봇) | `python-telegram-bot` (웹훅/폴링) | 🔧 추가 설정 필요 | 텔레그램 봇이 "삼성뉴스" 메시지를 수신하여 파이프라인 실행 | 텔레그램 메시지 "삼성뉴스" | 파이프라인 실행 트리거 |
| ⑪ | 중복 방지 | `notion-client` (DB 쿼리) | 🔧 추가 설정 필요 | 노션 DB에서 URL 기준 중복 체크 | 기사 URL | 중복 여부 (True/False) |
| ⑫ | 에러 알림 | `python-telegram-bot` | 🔧 추가 설정 필요 | ⑦과 동일 도구, 에러 메시지 전송 | 에러 내용 | 텔레그램 에러 알림 |
| ⑬ | 환경변수 관리 | `python-dotenv` | ✅ 즉시 사용 가능 | pip 설치만 필요 | .env 파일 | 환경변수 로드 |

## 2. 즉시 실행 가능한 단계

| 단계 | 작업 | 도구 | 비고 |
|------|------|------|------|
| ① | 기사 목록 수집 | requests + BeautifulSoup4 | pip install 후 바로 사용 |
| ② | 주제 필터링 | Python 내장 | 추가 설치 없음 |
| ③ | 폴백 판단 | Python 내장 | 추가 설치 없음 |
| ④ | 기사 본문 수집 | requests + BeautifulSoup4 | ①과 동일 |
| ⑤ | AI 요약 생성 | 로컬 AI Agent | 현재 세션에서 바로 수행 가능 |
| ⑥ | 메시지 조립 | Python 내장 | 추가 설치 없음 |
| ⑬ | 환경변수 관리 | python-dotenv | pip install 후 바로 사용 |

> **7개 단계가 즉시 실행 가능** — 핵심 파이프라인(수집→필터→요약→메시지 조립)이 모두 포함됩니다.

## 3. 추가 설정이 필요한 단계

### 3-A. 텔레그램 연동 (⑦⑩⑫)

| 설정 항목 | 상세 | 예상 소요 |
|-----------|------|-----------|
| `python-telegram-bot` 설치 | `pip install python-telegram-bot` | 1분 |
| 텔레그램 봇 토큰 확인 | BotFather에서 발급받은 토큰 → `.env`에 저장 | 이미 보유 |
| 채팅 ID 확인 | 봇에게 메시지 전송 후 getUpdates API로 확인 | 5분 |
| 수동 트리거 봇 서버 | 폴링 방식: `bot.polling()` 상시 실행 또는 launchd로 관리 | 30분 |

### 3-B. 노션 연동 (⑧⑪)

| 설정 항목 | 상세 | 예상 소요 |
|-----------|------|-----------|
| `notion-client` 설치 | `pip install notion-client` | 1분 |
| 노션 API 키 확인 | 노션 Integration 페이지에서 발급 → `.env`에 저장 | 이미 보유 |
| 노션 DB 생성 | 날짜/제목/요약/원문링크/해시태그/카테고리/수집일/실행모드 컬럼 | 10분 |
| DB ID 확인 | 노션 DB URL에서 추출 → `.env`에 저장 | 2분 |

### 3-C. 스케줄 실행 (⑨)

| 설정 항목 | 상세 | 예상 소요 |
|-----------|------|-----------|
| crontab 등록 | `crontab -e` → `0 8 * * * cd /path/to/project && python3 run_news_pipeline.py --mode auto` | 5분 |
| 또는 launchd plist | `~/Library/LaunchAgents/` 에 plist 파일 생성 | 10분 |

## 4. 그레이 영역

**그레이 영역 없음** — 모든 자동화 단계가 현재 사용 가능한 도구로 연결 가능합니다.

단, 아래는 향후 확장 시 고려할 수 있는 사항입니다:

```text
향후 확장 제안:
- 필요한 기능: 삼성전자 뉴스룸 RSS 피드 활용
- 왜 필요한가: 웹 스크래핑보다 안정적인 기사 수집 (사이트 구조 변경에 강함)
- 예상 입력: RSS 피드 URL
- 예상 출력: 구조화된 기사 목록
- 제안 스킬 이름: rss-news-collector
- 현재 상태: 뉴스룸에 RSS 피드가 있는지 확인 필요
```

## 5. 도구 연결 전 추가 확인 필요 항목

| 항목 | 현재 상태 | 해결 방법 |
|------|-----------|-----------|
| 삼성전자 뉴스룸 HTML 구조 | 미확인 | 오케스트레이션 실행 단계에서 실제 사이트 분석 |
| 생활가전 필터 키워드 최종 목록 | 초안 (생활가전, 냉장고, 세탁기, 에어컨, TV, 가전) | 실행하며 보완 |
| AI 요약 프롬프트 | 미작성 | 오케스트레이션 실행 단계에서 설계 |
| 텔레그램 수동 트리거 아키텍처 | 폴링 vs 웹훅 미확정 | 폴링 방식 권장 (로컬 환경, 별도 서버 불필요) |

> 위 항목들은 설계/인터뷰를 다시 할 필요 없이, **오케스트레이션 실행 단계에서 실제 구현하며 해결** 가능합니다.
