# 자동화 설계 초안: 삼성전자 뉴스룸 생활가전 뉴스 자동 브리핑

## 1. 목적

매일 아침 삼성전자 뉴스룸에서 생활가전 관련 최신 기사를 자동으로 수집하고, AI가 3줄 요약 + 해시태그로 정리하여 텔레그램으로 전달하고 노션 DB에 기록한다. 사람 개입 없이 완전 자동으로 동작한다.

## 2. 시작 트리거

### 2-A. 자동 실행 (스케줄)

| 항목 | 내용 |
|------|------|
| 실행 시간 | 매일 오전 8:00 |
| 실행 방법 | Mac `launchd` 또는 `cron` |
| 실행 명령 | `python3 run_news_pipeline.py --mode auto` |
| 수집 범위 | 전날 이후 최신 기사, 최대 5개 |

### 2-B. 수동 실행 (텔레그램 명령)

| 항목 | 내용 |
|------|------|
| 트리거 명령 | 텔레그램에서 **"삼성뉴스"** 입력 |
| 실행 방법 | 텔레그램 봇 웹훅 → `python3 run_news_pipeline.py --mode manual` |
| 수집 범위 | 최근 3일간 기사, 최대 5개 |

## 3. 입력 수집 및 검증

### 3-A. 입력 소스

| 항목 | 내용 |
|------|------|
| 소스 | 삼성전자 뉴스룸 웹사이트 |
| 수집 방식 | Python 웹 스크래핑 (requests + BeautifulSoup) |
| 수집 대상 | 기사 제목, 본문, 게시 날짜, 카테고리, URL |

### 3-B. 필터링 조건

| 순서 | 조건 | 설명 |
|------|------|------|
| 1 | 날짜 필터 | 자동: 전날 이후 / 수동: 최근 3일 |
| 2 | 주제 필터 | 생활가전 관련 (카테고리 또는 키워드 매칭) |
| 3 | 개수 제한 | 최대 5개, 최신순 정렬 |
| 4 | 폴백 | 생활가전 기사 0건 → 다른 카테고리 기사로 대체 |

### 3-C. 입력 검증

```text
검증 1: 뉴스룸 사이트 접속 가능한가? → 불가 시 3회 재시도 → 실패 시 "접속 불가" 알림
검증 2: 수집된 기사에 제목과 URL이 있는가? → 없으면 해당 기사 건너뛰기
검증 3: 중복 기사가 있는가? → URL 기준 중복 제거
```

## 4. 핵심 처리 단계

```text
[트리거] ──→ [① 기사 목록 수집] ──→ [② 주제 필터링] ──→ [③ 폴백 판단]
                                                              │
              ┌──────────────────────────────────────────────┘
              ▼
         [④ 기사 본문 수집] ──→ [⑤ AI 요약 생성] ──→ [⑥ 메시지 조립]
                                                          │
              ┌────────────────────────────────────────────┘
              ▼
         [⑦ 텔레그램 전송] ──→ [⑧ 노션 DB 기록] ──→ [완료]
```

### 단계별 상세

| 단계 | 작업 | 입력 | 출력 | 도구 | 환경변수 |
|------|------|------|------|------|----------|
| ① | 삼성전자 뉴스룸 기사 목록 수집 | 뉴스룸 URL + 모드(auto/manual) | 기사 목록 [{제목, URL, 날짜, 카테고리}] | requests + BeautifulSoup | — |
| ② | 생활가전 주제 필터링 | 기사 목록 | 필터링된 기사 목록 | 키워드 매칭 (생활가전, 냉장고, 세탁기, 에어컨, TV, 가전 등) | — |
| ③ | 폴백 판단 | 필터링 결과 | 최종 기사 목록 | 조건 분기 | — |
| ④ | 기사 본문 수집 | 기사 URL 목록 | 기사 본문 텍스트 목록 | requests + BeautifulSoup | — |
| ⑤ | AI 요약 생성 | 기사 본문 | 3줄 요약 + 해시태그 | LLM API (OpenAI 등) | `OPENAI_API_KEY` |
| ⑥ | 텔레그램 메시지 조립 | 요약 데이터 | 포맷팅된 메시지 문자열 | Python 문자열 처리 | — |
| ⑦ | 텔레그램 전송 | 메시지 문자열 | 전송 결과 | python-telegram-bot | `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID` |
| ⑧ | 노션 DB 기록 | 기사 데이터 + 요약 | DB 행 추가 결과 | notion-client | `NOTION_API_KEY`, `NOTION_DB_ID` |

### 조건 분기 상세

```text
[단계 ③ 폴백 판단]:
  조건 A: 생활가전 기사 1건 이상 → 그대로 진행
  조건 B: 생활가전 기사 0건 → 전체 카테고리에서 최신 기사 수집 + 메시지에 카테고리 표시

[단계 ⑤ AI 요약]:
  조건 A: 요약 성공 → 3줄 요약 + 해시태그 사용
  조건 B: 요약 실패 → 1회 재시도 → 재실패 시 기사 제목만 전달
```

## 5. 사람 검토 지점

**없음** — 완전 자동화(🟢)

수혜자는 텔레그램에서 결과 메시지를 확인하는 것으로 충분하다. 별도 승인/검토 단계 없이 수집 → 요약 → 전달 → 기록까지 자동으로 완료된다.

## 6. 저장 및 전달

### 6-A. 텔레그램 전달

**메시지 템플릿:**

```text
📰 삼성 뉴스룸 (생활가전) - YYYY/MM/DD

1. [기사 제목]
• 요약 포인트 1
• 요약 포인트 2
• 요약 포인트 3
#태그1 #태그2 #태그3
🔗 원문링크

2. [기사 제목]
• 요약 포인트 1
• 요약 포인트 2
• 요약 포인트 3
#태그1 #태그2 #태그3
🔗 원문링크

---
⏰ 자동 실행 | 📊 총 N건
```

**폴백 시 메시지 헤더:**

```text
📰 삼성 뉴스룸 (전체 — 생활가전 기사 없음) - YYYY/MM/DD
```

### 6-B. 노션 DB 기록

| 컬럼명 | 타입 | 내용 |
|--------|------|------|
| 날짜 | Date | 기사 게시 날짜 |
| 제목 | Title | 기사 제목 |
| 요약 | Text | 3줄 요약 |
| 원문링크 | URL | 기사 원문 URL |
| 해시태그 | Multi-select | #태그1, #태그2, ... |
| 카테고리 | Select | 생활가전 / 모바일 / 반도체 등 |
| 수집일 | Date | 파이프라인 실행 날짜 |
| 실행모드 | Select | auto / manual |

## 7. 예외 처리

### 7-A. 입력 단계 예외

| 예외 | 처리 방법 |
|------|-----------|
| 뉴스룸 사이트 접속 불가 | 3회 재시도 (10초 간격) → 실패 시 텔레그램으로 "⚠️ 뉴스룸 접속 불가" 알림 |
| 생활가전 기사 0건 | 다른 카테고리 최신 기사로 대체 + 메시지에 카테고리 명시 |
| 사이트 구조(HTML) 변경 | 스크래핑 실패 감지 → 텔레그램으로 "⚠️ 사이트 구조 변경 감지, 수동 확인 필요" 알림 |
| 기사 본문 로딩 실패 | 해당 기사 건너뛰고 나머지 처리 계속 |

### 7-B. 처리 단계 예외

| 예외 | 처리 방법 |
|------|-----------|
| AI API 호출 실패 | 1회 재시도 → 재실패 시 기사 제목 + 원문링크만 전달 (요약 생략) |
| AI API 키 만료/한도 초과 | 텔레그램으로 "⚠️ AI API 키 확인 필요" 알림 + 제목만 전달 |
| 처리 타임아웃 (60초) | 해당 기사 건너뛰기 |

### 7-C. 출력 단계 예외

| 예외 | 처리 방법 |
|------|-----------|
| 텔레그램 전송 실패 | 3회 재시도 → 실패 시 에러 로그 기록 |
| 노션 API 실패 | 텔레그램 전달은 정상 진행, 노션 실패만 로그 기록 |
| 메시지 길이 초과 (텔레그램 4096자 제한) | 기사 수를 줄여서 분할 전송 |

## 8. 중복 방지

| 대상 | 중복 기준 | 처리 방법 |
|------|-----------|-----------|
| 기사 수집 | 기사 URL | 동일 URL 기사 제거 |
| 노션 DB | 기사 URL | DB에 이미 존재하는 URL이면 기록 건너뛰기 |
| 텔레그램 | — | 중복 전송 방지는 노션 DB 체크로 간접 처리 |

## 9. 현재 자동화 가능한 부분

| 단계 | 자동화 가능 여부 | 비고 |
|------|------------------|------|
| ① 기사 목록 수집 | ✅ 가능 | requests + BeautifulSoup |
| ② 주제 필터링 | ✅ 가능 | 키워드 매칭 |
| ③ 폴백 판단 | ✅ 가능 | 조건 분기 |
| ④ 기사 본문 수집 | ✅ 가능 | requests + BeautifulSoup |
| ⑤ AI 요약 생성 | ✅ 가능 | OpenAI API 등 |
| ⑥ 메시지 조립 | ✅ 가능 | Python 문자열 처리 |
| ⑦ 텔레그램 전송 | ✅ 가능 | python-telegram-bot |
| ⑧ 노션 DB 기록 | ✅ 가능 | notion-client |
| 스케줄 실행 | ✅ 가능 | Mac cron/launchd |
| 수동 트리거 (텔레그램 봇) | ✅ 가능 | 텔레그램 봇 웹훅 + 로컬 서버 |

> 모든 단계가 현재 사용 가능한 도구로 자동화 가능합니다. 그레이 영역 없음.

## 📋 필요 Python 패키지

```text
requests          # HTTP 요청
beautifulsoup4    # HTML 파싱
openai            # AI 요약 API
python-telegram-bot  # 텔레그램 봇
notion-client     # 노션 API
python-dotenv     # 환경변수 관리
schedule          # (선택) 로컬 스케줄러
```

## 📋 필요 환경변수 (.env)

```text
OPENAI_API_KEY=           # AI 요약용 LLM API 키
TELEGRAM_BOT_TOKEN=       # 텔레그램 봇 토큰
TELEGRAM_CHAT_ID=         # 텔레그램 채팅 ID
NOTION_API_KEY=           # 노션 API 키
NOTION_DB_ID=             # 노션 데이터베이스 ID
SAMSUNG_NEWSROOM_URL=     # 삼성전자 뉴스룸 기본 URL
```
