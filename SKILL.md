# automation-pipeline-design-eduv-intermediate

## 이 스킬의 목적

이 스킬은 `중급` 수준에서 바로 적용할 수 있는 `6단계 자동화 흐름`을 다룹니다.

사용자에게 보이는 단계:

1. 인터뷰
2. 워크시트
3. 자동화 설계
4. 도구 연결
5. 오케스트레이션 실행 및 결과 확인
6. Agent 변환

이 스킬의 핵심 원칙:

- 만약 기존에 인터뷰 결과 및 워크시트가 있더라도 구체성이 상당히 부족할 것이라는 전제로 접근해야합니다. 빈틈을 파고들어 자세하게 추가 인터뷰 및 워크시트를 작성합니다.
- 인터뷰와 워크시트의 품질이 이후 자동화 품질을 결정합니다.
- 인터뷰는 스킬 내부 번들 인터뷰 가이드를 기준으로 반드시 `질문 1개씩` 진행합니다.
- 인터뷰의 핵심 항목이 비어 있으면 다음 단계로 넘어가지 않습니다.
- 사용자가 현재 답할 수 없다고 명시한 항목만 `[보류: 이유]`로 남깁니다.
- 현재 흐름과 무관한 과거 운영 체계나 다른 단계 설명을 사용자에게 꺼내지 않습니다.
- 복잡한 폴더 구조와 상태관리를 요구하지 않습니다.
- 현재 작업공간에서 실제로 찾은 `SKILL.md`를 먼저 확인합니다.
- 없는 기능은 억지로 있다고 가정하지 않고 `그레이 영역`으로 남깁니다.

이 스킬에서 하지 않는 것:

- 사용자가 이 스킬을 명시하지 않았는데 자동으로 이 스킬을 적용하는 것
- 얕은 인터뷰만 하고 바로 워크시트나 실행 단계로 점프하는 것
- 아직 없는 기능을 구현된 것처럼 설명하는 것
- 복잡한 운영 구조를 먼저 설계하는 것

## 언제 사용하는가

아래 경우에 이 스킬을 사용합니다.

- 사용자가 이 스킬 이름이나 경로를 직접 지정했을 때
- 사용자가 자동화 흐름을 질문형으로 따라가고 싶을 때
- 인터뷰에서 끝나지 않고 `도구 연결`, `실행 및 점검`, `Agent 변환`까지 보고 싶을 때
- 작업공간에 있는 스킬로 어디까지 자동화 가능한지 판단하고 싶을 때
- 없는 기능은 새 스킬 제작 후보로 정리하고 싶을 때

## 이 스킬의 의존 구조

이 스킬은 현재 작업공간 기준으로 필요한 참고 문서, 템플릿, 파일 검색 규칙을 내부에 포함한 `self-contained` 구성입니다.

### 스킬 내부에 포함된 것

- 단계 진행 규칙
- 질문 방식
- 템플릿 프롬프트
- 파일 검색 규칙
- 번들 참고 문서
- 작업공간 스킬 탐색 템플릿

참고 문서 경로:

```text
<skill_root>\references\
```

### 범위 내 자료 업데이트 원칙

- 이 스킬의 유지보수 범위는 `인터뷰 -> 워크시트 -> 자동화 설계 -> 도구 연결 -> 오케스트레이션 실행 및 결과 확인 -> Agent 변환`의 6단계입니다.
- 완전자동화와 관련해 추가 인터뷰 질문, 워크시트 항목, 업무정의/워크플로우 규칙, 툴 연결 방식, 오케스트레이션 실행 절차, 결과 확인 기준, Agent 변환 기준에 변경이 생기면 이 스킬이 참조하는 참고 문서, 템플릿, 예시 자료를 함께 업데이트해야 합니다.
- 특히 인터뷰와 워크시트 변경은 뒤 단계의 설계 품질에 직접 연결되므로, 선행 단계 문서가 바뀌면 자동화 설계, 도구 연결, 실행 프롬프트도 같이 확인합니다.

### 단계별 문서 매핑 원칙

기본 동작에는 외부 경로가 필요 없습니다.

- 내부 번들 참고 문서는 번호가 아니라 `문서 역할`로 찾습니다.
- `scripts/find_gpters_study_docs.py`는 유지하며, 각 단계 시작 전에 해당 단계용 매핑 결과를 다시 확인합니다.
- 문서 매핑 표의 backtick 역할명은 스크립트 내부 식별자이며, 사용자 설명에서는 역할 의미를 우선합니다.
- 단계 매핑에서 `NOT FOUND`가 나오면 누락 문서를 먼저 보고하고, 하드코딩된 번호로 강행하지 않습니다. 다만 현재까지 확보한 산출물과 보조 문서를 기준으로 다음 작업은 계속 진행합니다.
- 전체 역할 확인이 필요하면 기본 명령을, 현재 단계 정본만 좁혀 보고 싶으면 `--stage` 옵션을 사용합니다.

## 인터뷰와 워크시트 우선 원칙

- 이 스킬에서 가장 중요한 단계는 `인터뷰`와 `워크시트`입니다.
- 인터뷰는 단순 확인 절차가 아니라, 이후 자동화 설계와 도구 연결의 품질을 결정하는 핵심 단계입니다.
- 워크시트는 인터뷰 답변을 옮기는 문서이면서, 빠진 핵심 항목을 다시 검증하는 품질 게이트입니다.
- 후속 단계에서라도 핵심 정보가 비어 있다고 판단되면 인터뷰 또는 워크시트 단계로 되돌아갑니다.

## Step 0. 내부 참고 문서 확인

항상 아래 스크립트로 현재 번들 참고 문서 매핑을 먼저 확인합니다.

```powershell
python .claude\skills\_education_skills_중급\automation-pipeline-design-eduv-중급\scripts\find_gpters_study_docs.py
```

단계별 정본과 보조 문서를 좁혀 볼 때는 `--stage`를 사용합니다.

```powershell
python .claude\skills\_education_skills_중급\automation-pipeline-design-eduv-중급\scripts\find_gpters_study_docs.py --stage interview
```

필수 원칙:

- 숫자만 믿지 않습니다.
- 파일명의 `핵심 부분`으로 현재 문서를 찾습니다.
- 번호가 바뀌어도 같은 역할 문서를 찾아야 합니다.
- 기본 검색 대상은 스킬 내부 `bundled_study_docs`입니다.
- 각 단계 시작 전 해당 stage 매핑을 다시 확인합니다.
- `NOT FOUND`가 나온 역할은 누락 상태를 먼저 보고합니다.
- 정본 역할이 없더라도 하드코딩된 번호를 추정하지 않고, 현재까지 확보한 산출물과 보조 문서를 기준으로 단계 진행은 계속합니다.

이 단계는 `AI가 내부적으로 처리`하고, 사용자에게는 상세한 내부 체계를 설명하지 않습니다.

## Step 1. 의도 확인 (반드시 먼저 실행)

사용자가 이 스킬을 호출하면, 기존 프로젝트 유무와 관계없이 **먼저 사용자의 의도를 질문**합니다.

### 1-1. 기존 프로젝트 확인

`C:\501_aiworks\10_working\120autoproject\` 아래에 기존 자동화 프로젝트 폴더가 있는지 확인합니다.

### 1-2. 의도 확인 질문

기존 프로젝트가 **있는 경우**, 아래와 같이 질문 툴로 질문합니다:

```text
기존에 진행하던 자동화 프로젝트가 있습니다: [기존 프로젝트 목록]

지금 하고 싶은 것이 아래 중 어디에 해당하나요?

(A) 기존 [프로젝트명]의 남은 단계를 이어서 완료하고 싶다
(B) 기존 프로젝트와 별개로 새로운 자동화 업무를 설계하고 싶다
(C) 여러 자동화 파이프라인을 하나로 묶어서 오케스트레이션하고 싶다
(D) 위 중 복수 해당하거나, 다른 목적이 있다
```

기존 프로젝트가 **없는 경우**, 아래와 같이 질문합니다:

```text
자동화하고 싶은 업무가 있으신가요?
한 가지 업무만 자동화할 수도 있고, 여러 업무를 묶어서 진행할 수도 있습니다.
어떤 업무를 자동화하고 싶은지 간단히 알려주세요.
```

### 1-3. 금지 사항

- 기존 프로젝트가 있다고 해서 자동으로 해당 프로젝트를 이어가지 않습니다.
- 사용자의 답변을 듣기 전에 시작 단계를 결정하지 않습니다.
- 사용자가 여러 업무를 언급하면, 각 업무를 별도 파이프라인으로 분리할지 묶을지 추가 질문합니다.오직 하나의 업무 파이프라인씩 진행을 한다.

## Step 2. 시작점 판단

Step 1에서 사용자의 의도가 확인된 후, 해당 업무에 대해 이미 가진 자료에 따라 시작점을 정합니다.

| 현재 가지고 있는 것                 | 시작 단계                                 |
| ----------------------------------- | ----------------------------------------- |
| 아무것도 없음                       | 인터뷰부터 시작                           |
| 인터뷰 답변 메모 있음               | 워크시트부터 시작                         |
| 워크시트 있음                       | 자동화 설계부터 시작                      |
| 자동화 설계 초안 있음               | 도구 연결부터 시작                        |
| 도구 연결표 또는 관련 스크립트 있음 | 오케스트레이션 실행 및 결과 확인부터 시작 |

단, 앞 단계 정보가 부족하면 다음 문서를 쓰기 전에 질문으로 메웁니다.

- `인터뷰 답변 메모 있음`은 인터뷰 필수 항목이 대부분 채워져 있을 때만 해당합니다.
- 참여자/수혜자, 목적, 시작 조건, 입력, 출력, 예외, 사람 검토, 저장/전달 중 핵심 항목이 비어 있으면 워크시트가 아니라 인터뷰 보완부터 다시 진행합니다.

## Step 3. 단계별 실행 규칙

### 1. 인터뷰

- 기준 문서: 내부 `인터뷰` 참고 문서
- 사용 템플릿: `templates/01_interview_session_prompt.txt`
- 목표: 업무 목적, 시작 조건, 입력, 출력, 예외를 말로 드러냄
- 우선 원칙:
  - 단계 시작 전에 `python .claude\skills\_education_skills_중급\automation-pipeline-design-eduv-중급\scripts\find_gpters_study_docs.py --stage interview`로 현재 매핑을 다시 확인합니다.
  - 문서 매핑기에서 `interview` 역할로 찾은 현재 문서를 인터뷰의 사실상 정본으로 삼습니다.
  - `hub`, `flow_reference`, 사용 가이드 역할(`student_guide`)은 보조 참조로 사용합니다.
  - `interview`가 없으면 누락 상태를 안내하고, 보조 참조와 기존 산출물을 기준으로 인터뷰 진행은 계속합니다.
  - 질문은 반드시 한 번에 `1개`만 진행합니다.
  - 참여자/수혜자, 현재 고통, 자동화 범위, 시작 조건, 입력, 출력, 처리 흐름, 예외, 사람 검토 지점, 저장/전달, 실행 제약을 빠짐없이 확인합니다.
  - 사용자가 현재 답할 수 없는 항목만 `[보류: 이유]`로 남깁니다.
- 다음 단계 전환 조건:
  - 필수 인터뷰 항목이 채워졌거나, 충분히 질문한 뒤에도 현재 답할 수 없는 항목만 `[보류: 이유]`로 남긴 상태여야 합니다.

### 2. 워크시트

- 기준 문서: 내부 `워크시트` 참고 문서
- 사용 템플릿: `templates/02_worksheet_fill_prompt.txt`
- 목표: 인터뷰 결과를 구조화된 한 장의 워크시트로 정리
- 우선 원칙:
  - 단계 시작 전에 `python .claude\skills\_education_skills_중급\automation-pipeline-design-eduv-중급\scripts\find_gpters_study_docs.py --stage worksheet`로 현재 매핑을 다시 확인합니다.
  - 문서 매핑기에서 `worksheet`를 정본, `interview`를 보조 참조로 사용합니다.
  - `worksheet`가 없으면 누락 상태를 안내하고, 인터뷰 메모와 보조 참조를 기준으로 워크시트 초안 작성은 계속합니다.
  - 워크시트는 인터뷰 내용을 옮기는 단계이자, 빠진 핵심 항목을 다시 검증하는 단계입니다.
  - 워크시트 작성 중 핵심 빈칸이 발견되면 자동화 설계로 넘어가지 않고 인터뷰 보완 질문을 먼저 합니다.
  - 사용자가 끝까지 답할 수 없는 항목만 `[보류: 이유]`로 남깁니다.
- 다음 단계 전환 조건:
  - 워크시트의 핵심 섹션이 채워졌고, 남은 보류 항목에는 이유가 명시되어 있어야 합니다.

### 3. 자동화 설계

- 기준 문서: 내부 자동화 설계 참고 템플릿
- 사용 템플릿: `templates/03_workflow_rules_prompt.txt`
- 목표: 사용자가 이해할 수 있는 `자동화 설계 초안` 확보
- 단계 게이트:
  - 단계 시작 전에 `python .claude\skills\_education_skills_중급\automation-pipeline-design-eduv-중급\scripts\find_gpters_study_docs.py --stage automation_design`로 현재 매핑을 다시 확인합니다.

## 질문 방식

기본 규칙:

- 인터뷰 단계는 반드시 한 번에 `1개` 질문
- 워크시트 이후 단계는 한 번에 `3개 이하` 질문
- 현재 단계에 꼭 필요한 질문만
- 이미 앞 단계에서 답한 내용은 다시 묻지 않음
- 핵심 인터뷰 항목은 비워 둔 채 다음 단계로 넘기지 않음
- 충분히 질문했지만 사용자가 현재 답할 수 없다고 하면 `[보류: 이유]`로 표기하고 다음으로 진행 가능
- 비핵심 항목 또는 추후 검증 항목만 `[확인 필요]`로 둘 수 있음

질문 예:

- 인터뷰 단계: "이 업무 결과를 실제로 받아보는 사람은 누구인가요?"
- 인터뷰 단계: "이 업무에서 가장 자주 막히는 지점은 어디인가요?"
- 워크시트 단계: "워크시트 핵심 섹션 중 아직 비어 있는 항목은 무엇인가요?"
- 자동화 설계 단계: "중간에 사람이 검토해야 하는 지점이 있나요?"
- 도구 연결 단계: "현재 작업공간에서 이미 사용할 수 있는 스킬이 있나요?"
- 실행 단계: "지금 바로 실행할 수 있는 입력 자료가 준비되어 있나요?"

세부 규칙은 `references/03_interaction-protocol.md`를 봅니다.
