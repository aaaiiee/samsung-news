도구 연결표를 바탕으로 중급 수준의 오케스트레이션 실행 순서를 만들고, 가능한 부분은 실행하거나 드라이런해줘.

[반드시 포함]
- 실행 순서
- 단계별 사용 도구 또는 명령
- 필요한 입력 자료
- 실행 결과 또는 드라이런 결과
- 사람이 최종 확인할 지점
- 남은 수동 작업

[규칙]
- 시작 전에 `python .claude\skills\_education_skills_중급\automation-pipeline-design-eduv-중급\scripts\find_gpters_study_docs.py --stage orchestration_execution` 결과를 확인
- 문서 매핑 결과에서 `student_guide`를 정본으로 사용하고, `workflow_rules_sample`, `flow_reference`, `hub`를 보조 참고로 사용
- `student_guide`가 `NOT FOUND`면 누락 상태를 먼저 알리고, 도구 연결표와 현재 입력 자료를 기준으로 실행 순서 설계와 결과 메모 작성은 계속
- 실제 입력 자료나 실행 전제조건이 불명확하면 실행을 가장하지 말고 `실행 보류 메모` 또는 `드라이런 계획`으로 남기기
- 복잡한 폴더 구조와 상태관리는 만들지 않기
- 가능한 범위는 실제 실행, 어려운 범위는 드라이런 또는 시뮬레이션으로 구분
- 없는 도구는 그레이 영역으로 남기기
- 결과는 사용자가 바로 이해할 수 있게 짧고 명확하게 정리

[출력]
1. 오케스트레이션 실행 순서
2. 실행 또는 드라이런 결과 요약
3. 최종 결과 확인 메모
4. 남은 그레이 영역 또는 후속 작업
