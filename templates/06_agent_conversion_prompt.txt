오케스트레이션 실행 결과(orchestration_run.md)와 도구 연결표(tool_connection.md)를 바탕으로 이 자동화를 실제로 자율 실행할 수 있는 Agent 정의 파일을 만들어줘.

[반드시 포함]
- Agent 이름 (영문 소문자 하이픈 구분, 예: marketing-content-orchestrator)
- description: 언제 자동 호출되는지 트리거 조건 포함
- 실행 단계 (orchestration_run.md의 실행 순서를 그대로 반영)
- 각 단계에서 사용할 스킬/도구 명시
- sub-agent 분리가 필요한 단계 식별 (병렬 실행 가능 단계)
- 사람 개입 지점 명시 (Human-in-the-loop)
- 입력 트리거 조건 (파일 투입 / 명령어 / 스케줄)
- 출력 결과물 목록

[규칙]
- 시작 전에 `python .claude\skills\_education_skills_중급\automation-pipeline-design-eduv-중급\scripts\find_gpters_study_docs.py --stage agent_conversion` 결과를 확인
- 문서 매핑 결과의 사용 가이드 역할(`student_guide`), `workflow_rules_template`, 예시 참고 역할(`workflow_rules_sample`), `flow_reference`를 보조 참고로 사용
- .claude/agents/ 폴더에 저장하는 마크다운 파일 형식으로 작성
- 실제로 존재하는 스킬만 사용 (그레이 영역은 별도 표기)
- sub-agent가 필요하면 각각 별도 agent 파일로 분리 제안
- 복잡한 코드 없이 자연어 지시 기반으로 작성
- 없는 기능은 그레이 영역으로 남기고 새 스킬 후보 제안

[출력]
1. orchestrator agent 파일 (.claude/agents/{name}.md)
2. sub-agent 파일 목록 (필요한 경우)
3. Agent 변환 완료 체크리스트
4. 남은 그레이 영역 (자동화 불가 항목)
5. 실제 호출 테스트 방법 (검증 방법)
